# 模块 B：本地 PTY 终端详细设计

> Skill 绑定：code-architect（技术）+ frontend-design（前端）
> 架构决策：ADR-001（Socket.IO 统一传输）、ADR-003（事件总线）

## B.1 核心设计决策

1. **通过 Socket.IO `/pty` 命名空间通信** — 采用现有 Socket.IO 基础设施，添加新命名空间
2. **Socket.IO binary event 支持终端二进制流** — 终端输出和文件传输通过 Socket.IO binary emit 传输（ADR-001）
3. **使用 `@xterm/headless` + `@xterm/addon-serialize` 实现服务端 VirtualScreen** — 利用成熟的虚拟终端方案
4. **PTY 会话与现有 session/CLI 解耦** — 独立的注册表和 handler，不触碰现有 terminal proxy
5. **通过 shell 钩子追踪 CWD** — zsh 的 precmd_functions 和 bash 的 PROMPT_COMMAND 注入 OSC 序列
6. **Socket.IO 认证中间件** — `/pty` 命名空间强制 JWT 验证 + userId 归属校验（安全增强）
7. **PTY 进程资源限制** — rlimit 限制内存/CPU/FD，销毁时清理整个进程组（安全增强）

## B.2 通信协议

### 客户端 → 服务端事件

| 事件 | 载荷 | 说明 |
|------|------|------|
| `pty:create` | `{ ptyId, cols, rows, cwd? }` | 创建新 PTY 会话 |
| `pty:input` | `{ ptyId, data }` | 终端输入 |
| `pty:resize` | `{ ptyId, cols, rows }` | 调整终端大小 |

| 事件 | 载荷 | 说明 |
|------|------|------|
| `pty:ready` | `{ ptyId, shellType }` | PTY 创建成功 |

## B.7 关键实现说明

- **node-pty 兼容性**：需要原生编译，Bun 支持 node-pty 但可能需要构建配置
- **原子快照**：`snapshot()` 必须在互斥锁保护下执行，防止快照和广播数据重复
- **Scrollback 处理**：大 scrollback 应分块传输（参考成熟的分块传输方案）
- **与现有终端共存**：`/terminal` 命名空间保持不变，`/pty` 并行运行

## B.8 安全增强

### Socket.IO 认证中间件

```typescript
// /pty 命名空间认证（安全增强 N-1）
ptyNamespace.use(async (socket) => {
  const token = socket.handshake.auth.token
  if (!token) throw new Error('Authentication required')

  const payload = verifyJWT(token)
  socket.data.userId = payload.userId
  socket.data.sessionId = payload.sessionId
})

// 所有事件校验 userId 归属
function validatePtyOwnership(socket: Socket, ptyId: string): boolean {
  const session = ptyRegistry.get(ptyId)
  return session?.userId === socket.data.userId
}
```

### PTY 进程资源限制（安全增强 N-2）

```typescript
// 每个 PTY 进程设置 rlimit
interface PtyResourceLimits {
  memoryBytes: 536_870_912    // 512MB
  cpuSeconds: 3600            // 1 小时
  maxFileDescriptors: 256
}

// 销毁时清理整个进程组
function destroyPty(pty: IPtySession): void {
  try {
    process.kill(-pty.pid, 'SIGTERM')  // 负号表示进程组
  } catch {
    // 进程可能已退出
  }
}
```

### CWD 路径校验

`pty:create` 的 `cwd` 参数必须通过路径规范化检查，拒绝 `/`、`/etc`、`/root` 等敏感路径：

```typescript
const ALLOWED_CWD_ROOTS = [
  process.env.HOME,
  '/tmp',
  '/workspace',
]

function validateCwd(cwd: string): string {
  const resolved = path.resolve(cwd)
  if (!ALLOWED_CWD_ROOTS.some(root => resolved.startsWith(root))) {
    throw new Error('CWD not in allowed paths')
  }
  return resolved
}
```

## B.9 每用户限制

默认每个用户最多 8 个 PTY 会话，通过 `HAPI_POWER_PTY_MAX` 环境变量配置。全局 PTY 总量上限 256 个，防止资源耗尽。

## B.10 二进制帧集成（与 Module F.3 协作）

PTY 终端的输出/输入/快照等二进制帧通过 Socket.IO binary event 传输（ADR-001），无需额外 WebSocket 连接：

```typescript
// 服务端：发送终端二进制输出
socket.emit('pty:binary', Buffer.from([0x01, slot, ...payload]))

// 客户端：接收并分发
socket.on('pty:binary', (data: ArrayBuffer) => {
  const bytes = new Uint8Array(data)
  const opcode = bytes[0]
  if (opcode >= 0x01 && opcode <= 0x0F) {
    dispatchTerminalFrame(bytes)
  }
})
```
