# 模块 A：Git 管理详细设计

> Skill 绑定：code-architect（技术）+ frontend-design（前端）
> 架构决策：ADR-002（session 中心隔离）、ADR-004（Git 操作统一管理）

## A.1 核心设计决策

1. **Git 引擎使用 isomorphic-git + node:fs 后端** — Hub 服务器直接使用 isomorphic-git 操作本地文件系统，无需 CLI 协同代理
2. **Session 中心路由** — 路由嵌套在 `/api/sessions/:id/git/` 下，与平台 session 中心架构一致（ADR-002）
3. **采用现有认证中间件和 JWT 模式** — 无需新认证机制
4. **Git 凭证在 SQLite 中加密存储** — 使用 AES-256-GCM 加密（含 auth_tag），scope 限定在 namespace + userId
5. **长时间运行操作通过 operationId + 轮询模式** — clone/push/pull 返回 operationId，前端通过 TanStack Query 轮询状态
6. **Git 仓库通过 SQLite `git_repos` 表跟踪** — 提供仓库注册表，关联用户和 session，支持重启后恢复
7. **提供内部 API 供 Module E 调用** — autoCommit / getFileAtCommit / resetToCommit（ADR-004）

## A.3 数据库 Schema

```sql
CREATE TABLE IF NOT EXISTS git_repos (
    id TEXT PRIMARY KEY,
    session_id TEXT NOT NULL,            -- 关联 session（ADR-002）
    namespace TEXT NOT NULL,
    user_id INTEGER NOT NULL,
    name TEXT NOT NULL,
    clone_url TEXT NOT NULL,
    local_path TEXT NOT NULL,
    default_branch TEXT,
    current_branch TEXT,
    remote_name TEXT NOT NULL DEFAULT 'origin',
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_git_repos_session ON git_repos(session_id);

CREATE TABLE IF NOT EXISTS git_credentials (
    id TEXT PRIMARY KEY,
    namespace TEXT NOT NULL,
    user_id INTEGER NOT NULL,
    host TEXT NOT NULL,
    type TEXT NOT NULL DEFAULT 'https',
    username TEXT,
    encrypted_secret TEXT NOT NULL,
    iv TEXT NOT NULL,
    auth_tag TEXT NOT NULL,              -- GCM 认证标签（安全增强）
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX idx_git_creds_host_user ON git_credentials(host, user_id);
```

## A.5 错误处理策略

| 错误代码 | HTTP | 前端操作 |
|---------|------|---------|
| `auth_required` | 401 | 显示 GitAuthDialog |
| `conflict` | 409 | 显示冲突文件 |
| `network_error` | 502 | 显示重试横幅 |
| `dirty_workdir` | 409 | 提示暂存/提交 |
| `repo_exists` | 409 | 导航到已有仓库 |

## A.8 安全增强

1. **Clone URL SSRF 防护**：仅允许 `https://` 和 `ssh://` 协议，拒绝 `file://`、`git://`、`ftp://` 等协议
2. **凭证加密增强**：AES-256-GCM 加密包含 `auth_tag` 字段，防止密文篡改；密钥来源为环境变量 `HAPI_GIT_ENCRYPTION_KEY`，支持密钥轮换
3. **路径规范化**：所有 `local_path` 参数通过 `sanitizePath()` 中间件校验，防止路径遍历
4. **Rate Limiting**：clone/push/pull 等重操作限速 10 次/分钟/用户

## A.9 事件总线集成
