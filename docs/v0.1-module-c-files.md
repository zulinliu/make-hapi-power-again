# 模块 C：文件管理 + 代码编辑器详细设计

> Skill 绑定：code-architect（技术）+ ui-ux-pro-max（前端 UX）
> 架构决策：ADR-005（代码编辑器归入文件管理）

## C.1 核心设计决策

1. **使用 react-complex-tree** — 提供懒加载、拖放、虚拟化、内联重命名，比手写实现更可靠
2. **基于会话作用域的文件 API** — 保持与平台多机架构的一致性
3. **采用 Hono 路由模式** — 认证中间件 + Zod 验证 + runRpc 错误包装
4. **TanStack Query（服务器状态）+ 本地 state（UI 状态）** — 采用现有状态管理模式
5. **Monaco Editor 集成作为文件预览/编辑扩展** — 代码编辑器是文件管理的自然延伸（ADR-005）

### 拖放移动流程

1. 按下树项开始拖动 — 项目变半透明
2. 有效放置目标（目录）显示蓝色边框高亮
3. 放置后弹出 ConfirmDialog："将 'foo.ts' 移到 'src/utils/'？"
4. 放置在文件上 → 目标变为该文件的父目录
5. 完成后源和目标目录都失效并重新获取

### 搜索流程

1. Ctrl/Cmd+Shift+F 打开搜索栏
2. 300ms 防抖触发查询
3. 结果以扁平列表显示
4. 点击结果：选中文件、预览面板加载、树展开到父目录
5. Escape 关闭搜索

### 编辑器状态

```typescript
interface EditorState {
  filePath: string | null
  content: string
  language: string              // 从文件扩展名推断
  isDirty: boolean              // 有未保存变更
  isReadOnly: boolean           // 二进制/大文件时只读
  encoding: 'utf-8' | 'binary'
}
```

### Monaco Editor 集成要点

- **路由级懒加载**：Monaco Editor ~800KB gzip，使用 `React.lazy()` + `Suspense`
- **语言检测**：根据文件扩展名自动选择 Monaco language mode
- **自动保存**：可配置自动保存延迟（默认 2 秒），防止丢失
- **差异标记**：通过 `monaco.editor.getModel().getDecorationsInRange()` 标记 Git 变更行
- **大文件保护**：超过 1MB 的文件自动切换为只读 Shiki 预览
- **快捷键**：Ctrl+S 保存、Ctrl+P 快速打开文件、Ctrl+Shift+F 全局搜索

## C.10 安全增强

1. **路径规范化中间件**：所有文件操作路由使用 `sanitizePath()` 中间件，URL 解码 + Unicode NFC 正规化 + `fs.realpathSync()` 解析符号链接后再校验
2. **文件上传限制**：单文件最大 100MB，类型白名单（拒绝 .exe/.bat/.sh 等可执行文件），ZIP bomb 检测（压缩比 > 100:1 拒绝）
3. **下载安全**：ZIP 下载不包含符号链接，限制单次下载总大小 500MB
4. **搜索限制**：搜索深度最大 10 层，结果数最大 1000 条
