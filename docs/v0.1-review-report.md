## 一、CRITICAL 级别问题（必须修复）

### 1. 文件操作路径遍历漏洞（安全）

**来源**: 系统架构评审

文件管理 API 接受用户提供的 path、parentPath、sourcePath、destDir 参数，未做路径规范化检查。攻击者可通过 `../../../etc/passwd` 或符号链接跳转到工作目录之外。

**修复方案**: 在所有文件操作路由中添加路径规范化中间件：
```typescript
function sanitizePath(rootPath: string, userInput: string): string {
  const resolved = path.resolve(rootPath, userInput)
  if (!resolved.startsWith(rootPath + path.sep) && resolved !== rootPath) {
    throw new Error('Path traversal detected')
  }
  return resolved
}
```

### 2. 可访问性设计缺失（UX）

**来源**: UI/UX 评审

- 焦点环颜色 `#1E293B` 与背景 `#0F172A` 对比度极低，键盘用户无法看到焦点
- 缺少 ARIA 属性设计（上下文菜单、标签栏、列表分组）
- 终端模块缺少屏幕阅读器适配
- `--color-muted: #272F42` 作为文字色在暗色背景上对比度不足

**修复方案**:
- `--color-ring` 改为 `#22C55E`（accent 绿色）
- 新增 `--color-muted-text: #94A3B8`（对比度约 5.5:1）
- 为每个组件补充 ARIA 角色和标签设计要求

### 3. 大型依赖无懒加载策略（性能）

**来源**: 前端架构 + UI/UX 评审

Monaco Editor（~800KB gzip）、Mermaid.js（~1MB）、react-pdf（~200KB）、xterm.js（~500KB）均无 `React.lazy()` 动态导入规划。

**修复方案**: 在主设计文档中增加路由级代码分割策略：
```
/git/*          → 预加载 Git chunk（react-diff-view, Mermaid）
/workspace/*    → 预加载 Workspace chunk（Monaco, react-complex-tree, Shiki）
/terminal       → 预加载 Terminal chunk（xterm.js）
/settings/*     → 预加载 Extensions chunk
```

## 二、HIGH 级别问题（应修复）

### 4. 触摸交互无替代方案（UX）

文件管理器的右键菜单、拖放操作完全依赖鼠标和键盘快捷键，缺少长按触发、工具栏按钮替代等触摸适配。

**修复方案**: 为所有右键菜单操作添加工具栏入口，拖放增加"移动到..."对话框备选。

### 5. 插件沙箱隔离不足（安全）

Blob URL `import()` 在浏览器主线程执行，插件代码可访问 window、document、fetch 等全局 API。静态权限声明无法在运行时强制执行。

**修复方案**: v0.1 使用 ErrorBoundary 包裹每个插件面板 + 运行时权限检查网关；v0.2 评估 iframe sandbox 或 Web Worker 隔离方案。

### 6. node-pty + Bun 兼容性未验证（技术风险）

node-pty 是原生模块，依赖 N-API，与 Bun 的兼容性不确定。

**修复方案**: Phase 1 开始前创建 PoC 验证。备选方案：Bun.spawn + Unix PTY。

### 7. Git 模块绕过 session 隔离（架构一致性）

Git 模块使用独立 `/api/git/*` 路由，按 userId 隔离，与平台 session 中心架构不一致。

**修复方案**: git_repos 表增加 sessionPath 关联字段，支持多 CLI 实例场景。

### 8. 响应式布局策略缺失（UX）

四个模块均未定义断点体系。文件管理器左右分栏在小屏幕如何适配？PTY 终端在移动端的布局？

**修复方案**: 定义统一断点（640 / 1024 / 1440px），为每个模块提供每个断点的布局模式。

### 9. 无统一导航架构（UX）

四个功能模块的路由分散（/git, /local-terminal, /sessions/:id/files, /settings/extensions），缺少统一的侧边栏/底部导航。

**修复方案**: 设计统一侧边栏导航，整合 Chat / Git / Files / Terminal / Settings / Extensions。

## 三、MEDIUM 级别问题（建议修复）

### 10. 状态管理细节不足

- 缺少统一的 queryKey 工厂函数
- 关键操作缺少乐观更新策略
- 文件管理器剪贴板使用 Context 但未定义作用域

### 11. 错误响应格式不统一

Git 模块用自定义错误码表，文件模块用 runRpc 包装，插件模块未定义。

**修复方案**: 统一为 `{ success, error?: { code, message }, data? }` 格式。

### 12. 数据库迁移集成缺失

新表（git_repos, git_credentials）未集成到现有数据库迁移链。

### 13. API Rate Limiting 缺失

所有新增 API 路由无速率限制。

### 14. 动画和 reduced-motion 支持未设计

仅有一个过渡效果定义，缺少完整的动画规范和 `prefers-reduced-motion` 策略。

### 15. 空状态和错误状态设计不完整

仅文件管理器有 EmptyState，其他三个模块完全缺失。

### 16. Socket.IO 与 ws 混用

技术栈表列出 ws 库，但 PTY 模块实际使用 Socket.IO /pty 命名空间，存在冲突。

### 17. 模块间文件系统并发操作无协调

Git 和 Files 模块可同时操作同一文件系统路径，缺少路径级锁。

## 六、第二轮深度评审新增发现

### 新增 CRITICAL 问题

**N-1. PTY Socket.IO 命名空间认证缺失**

PTY 终端的 `/pty` 命名空间未设计连接认证。未授权用户可通过 `pty:reconnect` 接管其他用户的终端会话，获得远程 shell 访问权。`pty:create` 的 `cwd` 参数未做路径校验，可能以 `/` 作为工作目录启动 shell。

修复：Socket.IO middleware 验证 JWT，所有 PTY 事件校验 userId 归属，cwd 参数通过路径规范化检查。

**N-2. PTY 进程缺乏资源限制**

每用户 8 个会话限制不涉及进程级资源限制。fork bomb（`:(){ :|:& };:`）或 `cat /dev/urandom` 等命令可耗尽服务器资源。未定义全局 PTY 数量上限。进程销毁时未清理子进程树。

修复：每个 PTY 设置 rlimit（内存 512MB、CPU 3600s、FD 256），全局 PTY 上限，销毁时 kill 整个进程组。

**N-3. Git Clone URL 的 SSRF 攻击**

isomorphic-git 支持 `file://` 协议。攻击者可通过 `cloneUrl: "file:///etc"` 克隆服务器本地目录，再通过文件浏览读取 `/etc/shadow` 等敏感文件。

修复：Clone URL 白名单，仅允许 `https://` 和 `ssh://`，拒绝 `file://`、`git://`。

**N-4. 路径遍历防护不充分**

第一轮的 `sanitizePath` 仅用 `path.resolve + startsWith`，未覆盖：符号链接绕过、URL 编码绕过（%2e%2e%2f）、Unicode 等价字符、Windows 保留名称。

修复：增加 URL 解码 + Unicode NFC 正规化 + `fs.realpathSync` 解析符号链接后再校验。

### 新增 HIGH 问题

**N-5. Git 凭证加密密钥管理缺失**

AES-256-GCM 加密的密钥来源、轮换机制、IV 重用风险均未设计。缺少 `auth_tag` 字段。

**N-6. 多用户数据隔离不完整**

文件 API 通过 session ID 而非 userId 隔离，用户 A 可能通过猜测 session ID 访问用户 B 的文件。PTY 的 `get(ptyId)` 不校验 userId 归属。

**N-7. 文件上传缺乏安全控制**

无文件大小限制、类型限制、存储配额。ZIP 下载可能被利用为 zip bomb。

**N-8. 敏感数据日志泄露风险**

PTY 输入中的密码、Git 凭证明文、JWT token 可能被服务端日志记录。

### 新增架构缺口

**N-9. 代码编辑器模块完全缺失**

主文档架构图包含 CodeEdit/Preview，但无模块文档覆盖文件内容读写 API 和 Monaco Editor 集成。这是端到端流程的关键断点。

**N-10. 第一轮评审改进未穿透到模块文档**

主文档第 9 节声明的 7 项改进（路径安全、代码分割、错误格式、响应式断点等）未同步到 4 个模块文档中。实施团队按模块文档开发时将遗漏所有改进。

**N-11. 跨模块数据桥接缺失**

Git `local_path` 与 Files `sessionPath` 的路径映射、插件 PluginContext 到 PTY 服务的调用链、文件变更的跨模块事件通知均未设计。
