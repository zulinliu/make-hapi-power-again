# v0.18.0 截图与浏览器级 PWA 验收记录

## 固定顺序

1. 接入：模型星桥 / Model Nexus
2. 驾驶：引导光标 / Guide Beam
3. 观测：上下文脉冲 / Context Pulse
4. 追踪：Git 脉络 / Git Atlas
5. 沉淀：会话织锦 / Session Loom

## 生成方式

```bash
node scripts/generate-v018-screenshots.cjs
```

脚本会从 `web/` 启动真实 React 前端，固定访问 `http://127.0.0.1:53180`，用 Playwright 注入 mock API 和 SSE 数据后截图。mock 数据只使用 `example.com`、`git.internal.example.com`、`test-user`、`/home/tester/project` 等占位值，不包含真实 token、API key、私有 Git credential 或个人路径。

本轮成功输出：

```text
v0.18.0 screenshots generated and browser-level PWA checks passed.
```

## 浏览器级 PWA 验收

- 390×844 和 430×932 移动视口使用 iPhone Safari UA、touch、`prefers-reduced-motion: reduce`。
- PWA 安装提示通过 `pwa_install_dismissed` 抑制，避免遮挡 signature moment。
- Guide Beam 移动端检查 Composer 图标按钮、Stop、发送、定时发送、Guide 模式、队列编辑和队列取消的触控目标。
- Session Loom 移动端检查 Tab、面板按钮、select、label 的触控目标，并确认导出脱敏预览可见。
- Context Pulse 检查 `上下文：40%` 可点击打开详情，且阈值逻辑覆盖 59/60/80/81。
- 下载失败 copy/share fallback 已由 Phase 41 代码与测试覆盖；本轮浏览器截图覆盖导出预览和脱敏状态，不替代真实 iOS 下载行为验收。

## 非阻塞输出

- Vite dev 模式会出现 Service Worker registration error，这是开发服务器环境下的已知噪音，不代表 production PWA 注册失败。
- mock SSE / message 数据会触发 React duplicate key warning，这是截图脚本 mock 数据的重复渲染噪音，不作为生产缺陷记录。
