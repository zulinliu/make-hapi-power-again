# Hapi Power 前端重构设计方案

> 版本: v1.0 | 日期: 2026-06-05
> 覆盖范围: web/ 主应用全部页面、组件、弹窗、交互

---

## 一、现状诊断

### 1.1 当前技术栈
- React 19 + TanStack Router + TanStack Query
- Tailwind CSS 4 + class-variance-authority + Radix UI
- @assistant-ui/react 聊天框架 + xterm.js 终端 + Monaco 编辑器
- Workbox PWA (含 iOS 适配)
- oklch() 色彩系统

### 1.2 核心问题诊断

| 维度 | 问题 | 严重度 |
|------|------|--------|
| **视觉风格** | 偏工具化，缺乏品牌辨识度；蓝紫主色落入"AI产品"同质化窠臼 | 高 |
| **布局** | 侧边栏+主内容区分栏模式在移动端体验断裂，切换不够自然 | 高 |
| **排版** | DM Sans 可用但缺乏个性；Source Serif 4 作为 Display 字体在中文环境无表现力 | 中 |
| **色彩** | oklch 体系好但应用不彻底；暗色模式对比度不够；语义色缺乏层次 | 中 |
| **动效** | 仅有基础 transition-colors，缺乏入场动画、页面切换、微交互 | 高 |
| **移动端** | 简单的 `hidden lg:flex` 隐藏模式，未采用原生感的移动交互范式 | 高 |
| **组件** | 基础 UI 组件齐全但视觉精致度不够；空状态、骨架屏缺失 | 中 |
| **图标** | 全部内联 SVG，风格不统一，缺乏系统化管理 | 中 |
| **多端一致性** | 桌面和移动端使用完全不同的布局策略，视觉语言割裂 | 高 |

### 1.3 参考产品分析

| 产品 | 值得借鉴之处 |
|------|-------------|
| **Linear** | 命令面板(K)、流畅的页面转场、键盘优先的交互、极简侧边栏、移动端底部导航栏 |
| **Vercel Dashboard** | 深色为主的高对比设计、数据密集但层次分明、移动端卡片化布局 |
| **Arc Browser** | 侧边栏的空间利用、多工作区的概念、Command Bar 快捷操作 |
| **Notion** | 移动端 PWA 的原生感、底部工具栏、手势导航、表单的移动端优化 |
| **Figma** | 工具面板的收放逻辑、属性面板的分层、移动端查看器模式 |

## 二、设计方向

### 2.1 设计理念: "精密工具"

核心定位: Hapi Power 是开发者的精密工具，不是通用消费产品。

**三大原则:**
1. **Focus First** — 界面服务于工作流，减少视觉噪音。每个元素都有明确目的。
2. **Adaptive Consistency** — 不是"响应式缩放"，而是根据设备能力提供最优交互模式。桌面用宽屏效率，移动用触控原生感。
3. **Calm Power** — 平静但不无聊。深色模式为默认，用微妙的动效和质感表达品质感。

### 2.2 品牌色彩体系

```
全新主色: 深青墨 (Ink Teal)
  Primary:     oklch(52% 0.12 200)   — 深青，远离AI紫蓝
  Primary Hover: oklch(48% 0.14 200)
  Primary Subtle: oklch(52% 0.06 200 / 0.10)

强调色: 琥珀金 (Amber)
  Accent:      oklch(72% 0.16 75)    — 用于关键操作、状态高亮

语义色:
  Success:     oklch(65% 0.16 155)   — 翡翠绿
  Warning:     oklch(75% 0.15 80)    — 暖琥珀
  Danger:      oklch(60% 0.20 22)    — 珊瑚红
  Info:        oklch(65% 0.12 230)   — 钴蓝

中性色 (暖灰调):
  Light Canvas:  oklch(97.5% 0.005 75)   — 微暖白
  Dark Canvas:   oklch(12% 0.008 200)    — 带青调深黑
```

**设计决策说明:**
- 摒弃 AI 产品典型的紫蓝渐变，选择深青墨作为主色，传递"精密工具"气质
- 琥珀金作为强调色，在深色背景上极其醒目，用于关键 CTA
- 暖灰调中性色避免冷感，配合 PWA 的"原生应用"气质

### 2.3 字体方案

```
UI 字体:     Geist Sans (Variable)
  — Vercel 出品，几何感与人文感平衡，开发者工具标配
  — 替代: DM Sans (当前已有)

Display 字体: Geist Sans Semibold/Bold
  — 保持与 UI 字体同族，通过字重和大小建立层级
  — 不再使用 Source Serif 4 (中文环境无意义)

代码字体:    JetBrains Mono (Variable) (保留)
  — 开发者最熟悉的等宽字体，已在使用

中文字体:    system-ui 中文回退链
  — macOS: PingFang SC → Apple SD Gothic Neo
  — Windows: Microsoft YaHei
  — Android: Noto Sans CJK SC
  — 无需额外加载，使用系统字体
```

### 2.4 圆角策略

```
小组件 (Badge, Tag):     4px
输入框 (Input):          6px
按钮 (Button):           6px
卡片 (Card):             10px
弹窗 (Dialog):           14px
底部抽屉 (Bottom Sheet): 顶部 20px
移动端底部导航栏:         0px (全宽贴合)
```

### 2.5 阴影体系

```
Surface 0 (canvas):      无阴影
Surface 1 (raised):      0 1px 2px oklch(0% 0 0 / 0.03)
Surface 2 (card):        0 2px 8px oklch(0% 0 0 / 0.06)
Surface 3 (dropdown):    0 4px 16px oklch(0% 0 0 / 0.10)
Surface 4 (dialog):      0 8px 32px oklch(0% 0 0 / 0.15)
Overlay:                 0 16px 48px oklch(0% 0 0 / 0.25)

注: 暗色模式下阴影改为发光效果
  Surface 2 (dark):      0 2px 8px oklch(52% 0.06 200 / 0.08)
  Surface 3 (dark):      0 4px 16px oklch(52% 0.08 200 / 0.12)
```

### 2.6 动效体系

```
入场动画:
  页面切换:     fade + slide (y: 8px)  200ms ease-out
  元素入场:     staggered fade-in       80ms 间隔, 150ms each
  弹窗入场:     scale(0.96) → 1 + fade 200ms cubic-bezier(0.16, 1, 0.3, 1)

交互动效:
  按钮悬停:     background-shift        120ms
  按钮按下:     scale(0.97)             80ms
  列表项悬停:   background-highlight     120ms
  侧边栏展开:   width transition         200ms ease-out
  Tab切换:      下划线滑动 + content fade  200ms

移动端动效:
  底部Sheet:    transform translateY + backdrop  300ms spring(1, 0.8)
  页面转场:     iOS风格 slide-in-from-right      300ms
  下拉刷新:     custom pull indicator

减速动效:
  @media (prefers-reduced-motion: reduce):
    所有动画 → opacity only, duration ≤ 100ms
```

### 3.3 移动端布局 (<768px) — PWA原生感

```
┌─────────────────────────┐
│ 安全区顶部 (env(safe-area-│
│ inset-top))              │
├─────────────────────────┤
│ 导航栏 (56px)            │
│ [← 返回] 页面标题 [···]  │
├─────────────────────────┤
│                         │
│    主内容区域             │
│    (全宽, 无侧边栏)       │
│                         │
│                         │
│                         │
├─────────────────────────┤
│ 底部安全区域              │
│ env(safe-area-inset-     │
│ bottom)                  │
└─────────────────────────┘

移动端专属:
  - 会话列表页: 全屏列表 + 浮动 "+" 按钮
  - 会话详情页: 全屏聊天 + 底部Composer
  - 子页面: iOS 风格的从右滑入转场
  - 返回手势: 支持 iOS 边缘滑动返回

iOS PWA 关键适配:
  1. viewport-fit=cover + safe-area-inset-padding
  2. -webkit-overflow-scrolling: touch
  3. touch-action: manipulation (禁止双击缩放)
  4. status bar: black-translucent
  5. 输入框聚焦时不缩放 (font-size ≥ 16px)
  6. 底部安全区域内不放置可点击元素
  7. 长按菜单自定义 (禁止默认选择/复制菜单)
```

## 十三、设计验收标准

1. **视觉一致性**: 所有页面使用统一的设计令牌，无硬编码颜色/间距
2. **多端适配**: 桌面(≥1024px) / 平板(768-1023px) / 手机(<768px) 三端体验完整
3. **iOS PWA**: 作为独立App使用时无Web痕迹（无缩放、无弹性过度、安全区域完整）
4. **暗色模式**: 所有页面暗色模式下对比度 ≥ 4.5:1 (WCAG AA)
5. **交互完整**: 每个可交互元素有 hover/active/focus/disabled 状态
6. **加载状态**: 所有异步数据加载使用骨架屏
7. **空状态**: 所有列表/数据展示有空状态设计
8. **动效克制**: 总体动效≤300ms, 遵循 prefers-reduced-motion
9. **无障碍**: focus ring 可见, ARIA 标签完整, 键盘可导航
10. **性能**: 无未使用的字体加载, CSS ≤ 50KB gzipped, 首屏 ≤ 1.5s
