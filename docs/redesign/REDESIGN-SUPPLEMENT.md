## A. CSS 变量迁移映射

### A.1 --app-* 到 --hp-* 迁移策略

**迁移原则:**
1. `--hp-*` 作为新的设计令牌源头，定义在 `tokens.css`
2. `--app-*` 作为语义别名层，引用 `--hp-*`，保留向后兼容
3. Telegram `--tg-theme-*` 覆盖通过 `--app-*` 别名层透传
4. 逐步迁移: 先更新 `--app-*` 的值为 `--hp-*` 引用，组件代码无需改动

```css
/* 迁移后的 index.css 结构 */

  --app-bg: var(--tg-theme-bg-color, var(--hp-canvas));
```

### A.2 Telegram Mini App 平台支持

```
Telegram 适配策略:
  1. --app-* 变量的 var(--tg-theme-*, fallback) 回退链保留
  2. data-telegram-app="true" 时的特殊处理保留:
     - overflow: hidden
     - tg-viewport-stable-height
  3. Telegram BackButton 集成保留
  4. Telegram 触觉反馈保留
  5. Telegram 主题色会覆盖 --app-* 的 hp-* 默认值
  6. 新增: Telegram 环境下使用 TG 的 primary_color 替代 --hp-primary

Telegram 专属 UI 调整:
  - 隐藏返回按钮 (Telegram 提供 BackButton)
  - 隐藏 PWA 安装提示
  - 使用 TG 主题色渲染导航栏
  - Bottom Sheet 背景使用 TG 的 secondary_bg_color
```

## D. 无障碍设计规范

### D.1 键盘导航

```
全局快捷键:
  Ctrl/Cmd + K    命令面板 (未来)
  Ctrl/Cmd + N    新建会话
  Ctrl/Cmd + /    搜索
  Escape          关闭弹窗/取消操作

焦点管理:
  弹窗打开 → 焦点移入弹窗, 焦点陷阱
  弹窗关闭 → 焦点返回触发元素
  Tab 切换 → 遵循视觉顺序 (左→右, 上→下)
  页面切换 → 焦点移入主内容区

焦点指示器:
  所有可交互元素: 2px Primary ring + 2px offset
  焦点可见性: 始终显示 (不使用 focus-visible-only)

跳过导航:
  <a href="#main-content" class="skip-link">跳到主内容</a>
  skip-link: 绝对定位, 正常隐藏, Tab 时显示
```

### D.2 ARIA 标注

```
角色标注:
  导航: <nav aria-label="会话导航">
  主内容: <main id="main-content">
  侧边栏: <aside aria-label="会话列表">
  搜索: role="search"
  Tab列表: role="tablist" + role="tab" + role="tabpanel"
  对话框: role="dialog" + aria-modal="true" + aria-labelledby

实时区域:
  聊天消息: aria-live="polite"
  Toast: role="status" aria-live="polite"
  错误: role="alert" aria-live="assertive"
  加载状态: aria-busy="true"
  进度: role="progressbar" + aria-valuenow

代码块标注:
  <pre role="region" aria-label="代码: {language}">
  <code aria-label="{language} 代码">

差异查看器:
  aria-label="文件差异: {filename}"
  添加行: aria-label="添加: {line content}"
  删除行: aria-label="删除: {line content}"
```

### D.3 对比度验证

```
必须验证的对比度对:

亮色模式:
  Text-primary (#~252530) on Canvas (#~F8F7F5):     预计 > 12:1 ✓
  Text-secondary (#~6B6E75) on Canvas:              预计 ~5.5:1 ✓
  Text-tertiary (#~9A9DA5) on Canvas:               预计 ~3.5:1 ✗ (仅装饰性文字)
  Primary (#~357E8E) on Canvas:                      需验证 ≥ 4.5:1
  Primary-text (white) on Primary:                    预计 > 6:1 ✓
  Danger on Canvas:                                   需验证 ≥ 4.5:1

暗色模式:
  Text-primary (#~F0F2F5) on Canvas (#~1A1C20):     预计 > 14:1 ✓
  Text-secondary (#~A0A5AE) on Canvas:               预计 ~7:1 ✓
  Primary (#~5AAABB) on Canvas (#~1A1C20):           需验证 ≥ 4.5:1
  琥珀强调 on 暗色:                                   需验证

大文本 (≥18px 或 ≥14px bold): 对比度要求 ≥ 3:1
普通文本: ≥ 4.5:1 (WCAG AA)
增强级: ≥ 7:1 (WCAG AAA, 可选目标)
```

### D.4 色盲友好

```
状态色不能仅依赖颜色区分，必须配合:
  成功: 绿色 + ✓ 勾号图标
  警告: 黄色 + ⚠ 三角图标
  危险: 红色 + ✕ 叉号图标
  信息: 蓝色 + ℹ 图标

Git 状态:
  Added: 绿色 + "A" 文字
  Modified: 黄色 + "M" 文字
  Deleted: 红色 + "D" 文字
  Untracked: 灰色 + "?" 文字

差异:
  添加行: 绿色背景 + "+" 行号
  删除行: 红色背景 + "-" 行号
```
