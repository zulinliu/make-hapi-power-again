# Plan v9.3-01: useContextMenu Hook + Git Status Panel Context Menu

> Create the `useContextMenu` hook and add right-click + "..." button context menu to GitStatusPanel file rows.

## Goal

Add right-click context menu support to Git status panel file rows with a reusable `useContextMenu` hook that wraps `useLongPress`. The menu provides View Diff, Copy Path, and Open File actions. Each file row also gets a visible "..." button for mobile-friendly access.

## Context

- Decisions: D-01 (useContextMenu hook), D-02 (Git menu items: Diff + Path + Open), D-03 (Git file rows get "..." button)
- Requirements: CTXMENU-01, CTXMENU-05, CTXMENU-06
- CTXMENU-02 and CTXMENU-03 already handled by existing DirectoryTree + useLongPress
- CTXMENU-04 already handled by v9.2 (ContextMenu uses var(--app-*))
- Existing: `useLongPress` at `web/src/hooks/useLongPress.ts`, `ContextMenu` at `web/src/components/ui/ContextMenu.tsx`
- Reference: `files.tsx` line 379-462 for context menu items pattern and clipboard copy pattern

Create a thin wrapper around `useLongPress` with a simplified API:

```typescript
interface ContextMenuPosition {
    x: number
    y: number
}

export function useContextMenu(
    onContextMenu: (position: ContextMenuPosition) => void
): {
    onMouseDown: React.MouseEventHandler
    onMouseUp: React.MouseEventHandler
    onMouseLeave: React.MouseEventHandler
    onTouchStart: React.TouchEventHandler
    onTouchEnd: React.TouchEventHandler
    onTouchMove: React.TouchEventHandler
    onContextMenu: React.MouseEventHandler
}
```

Implementation:
1. Import `useLongPress` from `./useLongPress`
2. Create a stable callback ref for `onContextMenu`
3. Call `useLongPress` with the callback, passing through `({ x, y }) => onContextMenu({ x, y })`
4. Return the handlers object from useLongPress

**2d. Apply useContextMenu to file rows**

Each file row (`<div key={file.path}>`) gets:
- Spread handlers from `useContextMenu` for right-click/long-press
- A "..." button at the end of the row

The file row structure changes from:

```tsx
<div key={file.path} className="flex items-center gap-2 text-sm py-1 px-2 rounded bg-[var(--app-secondary-bg)]">
```

To:

```tsx
<div
    key={file.path}
    {...menuHandlers(file)}
    className="flex items-center gap-2 text-sm py-1 px-2 rounded bg-[var(--app-secondary-bg)] cursor-pointer hover:bg-[var(--app-subtle-bg)] transition-colors"
>
    <GitStatusBadge status={file.status} />
    <span className="font-mono text-xs truncate flex-1 text-[var(--app-fg)]">
        {file.path}
    </span>
    <button
        type="button"
        className="flex items-center justify-center w-6 h-6 rounded text-[var(--app-hint)] hover:text-[var(--app-fg)] hover:bg-[var(--app-subtle-bg)] transition-colors shrink-0"
        onClick={(e) => {
            e.stopPropagation()
            const rect = e.currentTarget.getBoundingClientRect()
            setContextMenu({ x: rect.left, y: rect.bottom + 4, path: file.path, status: file.status })
        }}
    >
        ···
    </button>
</div>
```

For the hook, create a per-row handler factory since we need to capture `file.path` and `file.status`:

```typescript
const handleContextMenu = useCallback((position: { x: number; y: number }, file: GitFile) => {
    setContextMenu({ ...position, path: file.path, status: file.status })
}, [])
```

Note: Since `useContextMenu` returns handlers and we need per-file context, use `useLongPress` directly with a callback factory pattern, OR use a wrapper component for each row.

**Recommended approach:** Create a small `GitFileRow` inner component that uses `useContextMenu` internally for its specific file:

```tsx
function GitFileRow({ file, onContextMenu, onViewDiff, onCopyPath, onOpenFile, t }: {
    file: GitFile
    onContextMenu: (pos: { x: number; y: number }, file: GitFile) => void
    onViewDiff?: (path: string) => void
    onCopyPath?: (path: string) => void
    onOpenFile?: (path: string) => void
    t: (key: string) => string
}) {
    const handlers = useContextMenu((pos) => onContextMenu(pos, file))

    return (
        <div
            {...handlers}
            className="flex items-center gap-2 text-sm py-1 px-2 rounded bg-[var(--app-secondary-bg)] cursor-pointer hover:bg-[var(--app-subtle-bg)] transition-colors"
        >
            <GitStatusBadge status={file.status} />
            <span className="font-mono text-xs truncate flex-1 text-[var(--app-fg)]">
                {file.path}
            </span>
            <button
                type="button"
                className="flex items-center justify-center w-6 h-6 rounded text-[var(--app-hint)] hover:text-[var(--app-fg)] hover:bg-[var(--app-subtle-bg)] transition-colors shrink-0"
                onClick={(e) => {
                    e.stopPropagation()
                    const rect = e.currentTarget.getBoundingClientRect()
                    onContextMenu({ x: rect.left, y: rect.bottom + 4 }, file)
                }}
            >
                ···
            </button>
        </div>
    )
}
```

**3a. Wire callbacks in git.tsx**

In the git.tsx route where `GitStatusPanel` is rendered, pass the new callback props:

1. `onViewDiff`: Navigate to the diff file view using TanStack Router `navigate`:
   ```typescript
   const handleViewDiff = useCallback((path: string) => {
       navigate({
           to: '/sessions/$sessionId/file',
           params: { sessionId },
           search: { path: encodeBase64(path) }
       })
   }, [sessionId, navigate])
   ```

2. `onCopyPath`: Copy full path to clipboard:
   ```typescript
   const handleCopyPath = useCallback(async (path: string) => {
       const basePath = session?.metadata?.path ?? ''
       const fullPath = basePath ? `${basePath}/${path}` : path
       const ok = await copyToClipboard(fullPath)
       if (ok) addToast({ title: t('git.context.copyPathSuccess'), body: '' })
   }, [session?.metadata?.path, t])
   ```

   Import `copyToClipboard` from `@/lib/clipboard` (or wherever files.tsx gets it from).

3. `onOpenFile`: Navigate to file editor:
   ```typescript
   const handleOpenFile = useCallback((path: string) => {
       navigate({
           to: '/sessions/$sessionId/file',
           params: { sessionId },
           search: { path: encodeBase64(path) }
       })
   }, [sessionId, navigate])
   ```

Note: `onViewDiff` and `onOpenFile` navigate to the same route but with different context. View Diff goes to the diff view (staged/unstaged changes), Open File goes to the file editor. Since the file route already handles both based on `staged` search param, both callbacks use the same navigation pattern but `onViewDiff` is for diff context and `onOpenFile` is for editing context.
