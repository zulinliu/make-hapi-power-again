# Discussion Log — v9.3 Context Menu Adaptation

## Gray Areas Identified

1. **Hook 封装策略** — How to wrap useLongPress for simpler consumption
2. **Git 文件右键菜单项** — What actions for Git file right-click menu
3. **Git 文件行"..."按钮** — Whether to add visible "..." button on Git file rows

## Discussion Results

### G-01: Hook 封装策略

**Options presented:**
1. 直接用 useLongPress — No wrapper, consumers import useLongPress directly
2. 新包装 useContextMenu — Thin wrapper with simpler API
3. 统一到组件层 — Create a ContextMenuTrigger wrapper component

**User chose:** Option 2 — 新包装 useContextMenu

**Rationale:** useLongPress returns 8 handlers which is verbose. A thin wrapper accepting just an `onContextMenu` callback makes consumption much cleaner. The "..." button stays as independent logic.

### G-02: Git 文件右键菜单项

**Options presented:**
1. 最小集 — View Diff + Copy Path only (no backend work needed)
2. 中等集 — View Diff + Copy Path + Open File (uses existing APIs)
3. 完全集 — All actions including discard/stage (needs new backend APIs)

**User chose:** Option 2 — 中等集

**Rationale:** Open File navigation uses existing readSessionFile API. Discard and stage operations need new backend endpoints that are out of scope for v9.3.

### G-03: Git 文件行"..."按钮

**Options presented:**
1. 仅右键/长按 — No visible button, context menu via right-click/long-press only
2. 加 "..." 按钮 — Add visible "..." button for mobile-friendly access

**User chose:** Option 2 — 加 "..." 按钮

**Rationale:** Mobile users cannot right-click. Consistent with file manager UX. Small visual complexity increase is acceptable for better accessibility.

## Decisions Summary

| ID | Decision | Choice |
|----|----------|--------|
| D-01 | Hook 封装策略 | 新包装 useContextMenu |
| D-02 | Git 菜单项 | 中等集 (Diff + Path + Open) |
| D-03 | "..." 按钮 | 加 "..." 按钮 |
