# Phase v9.3 — Right-Click Context Menu Global Adaptation

## Scope

Add right-click context menu support to all sub-page file lists, starting with GitStatusPanel. Create a reusable useContextMenu hook for consistent desktop right-click + mobile long-press behavior.

## Decisions

### D-01: Hook 封装策略 — 新包装 useContextMenu

Create a new `useContextMenu` hook that wraps the existing `useLongPress` hook with a simpler API.

- **Interface:** `useContextMenu(onContextMenu: (pos: { x: number; y: number }) => void) => handlers`
- **Returns:** spreadable event handlers for any element
- **Wraps:** `useLongPress` (which already handles `onContextMenu` + touch events)
- **"..." button:** stays as independent trigger logic, NOT unified into the hook
- **File:** `web/src/hooks/useContextMenu.ts`

**Why:** useLongPress already works correctly for right-click and long-press, but its API is verbose (returns 8 handlers). A thin wrapper gives consumers a single function call.

**How to apply:** Any component needing right-click context menu imports `useContextMenu` instead of `useLongPress` directly. The "..." button on DirectoryTree rows stays as-is — it calls `onContextMenu` prop directly.

### D-02: Git 文件右键菜单项 — 中等集

Git status panel file right-click menu includes 3 actions:

1. **查看 Diff** — Navigate to diff view using existing `handleOpenFile` pattern from files.tsx
2. **复制路径** — Copy file path to clipboard (reuse pattern from files.tsx)
3. **打开文件** — Navigate to file editor using `readSessionFile` API (existing)

**NOT included in v9.3 scope** (no backend API):
- Discard changes (needs `git checkout/restore`)
- Stage/unstage files (needs `git add/reset`)
- Delete files

**Why:** Backend has diff and file read APIs ready. Stage/unstage/discard require new API endpoints — too much scope for v9.3.

**How to apply:** GitStatusPanel receives an `onFileAction` callback prop or similar. The parent (git.tsx) wires navigation and clipboard logic. Menu items reference i18n keys `git.context.*`.

### D-03: Git 文件行"..."按钮 — 加

Each file row in GitStatusPanel gets a visible "..." button at the end of the row.

- **Position:** right-aligned, same style as DirectoryTree "..." button
- **Trigger:** opens the same context menu as right-click/long-press
- **Mobile:** provides accessible touch target for menu trigger

**Why:** Mobile users cannot right-click. The "..." button ensures menu access on all devices, consistent with file manager behavior.

**How to apply:** Add "..." button to each file row in GitStatusPanel. Button triggers the same context menu. Use `useContextMenu` on the row for right-click, "..." button for tap.
