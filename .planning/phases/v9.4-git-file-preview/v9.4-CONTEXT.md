# Phase v9.4 — Git 状态文件预览

## Scope

在 Git Status tab 中，点击变更文件在侧面板（桌面端）或底部 Sheet（移动端）中预览文件完整内容，支持 Shiki 代码高亮。预览面板显示文件路径和变更状态，可直接跳转到文件管理页面。v9.3 的右键菜单 View Diff / Open File 仍保留（导航到文件路由）。

## Decisions

### D-01: 预览触发方式 — 单击文件行直接预览

单击 Git 文件行直接在 Git 页面内打开预览面板。右键菜单的 View Diff 和 Open File 保留导航到文件路由的行为不变。

- **Why:** 单击是最直观的交互方式。右键菜单操作（v9.3 已有）和单击预览是两个不同层级的交互。
- **How to apply:** GitFileRow 的单击事件触发 onPreview 回调，不触发 onClick（useContextMenu 已设 onClick: undefined）。预览面板由 git.tsx 管理 state。

### D-02: 容器形态 — 侧面板 slide-over（桌面端） / 底部全屏 Sheet（移动端）

桌面端（≥768px）: 右侧 slide-over 面板，不完全遮挡文件列表，宽度约 50-60%。
移动端（<768px）: 从底部滑出的全屏 Sheet，最大化预览空间。

- **Why:** 桌面端有足够空间做侧面板同时查看列表和预览。移动端空间有限，底部 Sheet 是 iOS/Android 原生模式的预览交互。
- **How to apply:** 新建 GitFilePreview 组件，响应式布局。使用 `md:` 断点切换两种形态。面板有关闭按钮（X）和拖拽关闭手势。

### D-03: 预览内容 — 文件完整内容 + Shiki 代码高亮

预览面板使用 `readSessionFile` API 读取文件内容，用 `CodeBlock` 组件（内置 Shiki 高亮）渲染。

- **Why:** CodeBlock 已有 Shiki 集成、行号、复制按钮、折叠长内容等能力。对于变更文件，看完整内容比只看 diff 更直观。
- **How to apply:** 导入 `CodeBlock` 组件，传入文件内容和解码后的语言类型。检测语言基于文件扩展名（复用 `langAlias` 映射）。

### D-04: 特殊文件处理

- **已删除文件（status === 'D'）:** 显示提示"文件已删除"而非预览内容
- **未跟踪文件（status === '?'）:** 正常预览（readSessionFile 仍可用）
- **二进制文件:** 检测后显示"二进制文件，无法预览"提示
- **图片文件:** 显示 ImagePreview（已有组件）

### D-05: 预览面板功能

预览面板顶部区域包含：
- 文件路径（完整路径，font-mono）
- 变更状态徽章（复用 GitStatusBadge）
- 关闭按钮（X）
- "在文件管理中打开" 按钮（跳转到文件路由）

### Claude's Discretion

- Slide-over 具体宽度（建议 50% ~ 60vw）
- 动画过渡效果（建议 transform + transition）
- 加载状态展示
- 错误状态展示
