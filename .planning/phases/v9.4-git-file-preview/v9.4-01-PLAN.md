<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> API (readSessionFile) | File content request crosses trust boundary; server validates path scoping |
| API response -> Client rendering | Base64 decoded content rendered as text (CodeBlock) or data URI (images) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-v9.4-01 | Tampering | readSessionFile path param | mitigate | Server-side path validation already enforces sessionId scoping -- no action needed |
| T-v9.4-02 | Tampering (XSS) | File content rendered in preview | mitigate | CodeBlock renders as text with Shiki highlighting (no dangerouslySetInnerHTML for user content). ImagePreview uses img element with data URI. MarkdownFilePreview uses react-markdown (no raw HTML). |
| T-v9.4-03 | Denial of Service | Large file preview rendering | accept | CodeBlock has collapseLongContent prop for visual truncation. Shiki processes in WASM and is reasonably fast. For extremely large files (>500KB), consider adding a size check in a future iteration. Low-value target for a single-user tool. |
</threat_model>
