# Plan v9.2-02: Migrate All Sub-Pages to SubPageLayout

> Wrap all 5 sub-pages (git, extensions, changes, timeline, undo) in the shared SubPageLayout component, converting pill buttons to underline tabs where applicable.

## Goal

Every sub-page route uses SubPageLayout for consistent layout: max-w-content content wrapper, unified tab underline indicator, toolbar slot for page actions. Pill-style filter buttons in changes/timeline/undo are converted to underline tabs (or kept as form controls in undo's case). After this plan, all sub-pages share the same visual skeleton.

## Context

- Decisions: D-05 through D-11 (SubPageLayout component, tab style, pill-to-tab conversion, content width)
- Requirements: LAYOUT-02 (max-w-content), LAYOUT-03 (safe area), LAYOUT-04 (back button), LAYOUT-05 (underline tab), LAYOUT-06 (p-3 padding), LAYOUT-08 (unified pages)
- Depends on: Plan v9.2-01 must complete first (SubPageLayout component and CSS variables must exist)
- Read first: `web/src/components/ui/SubPageLayout.tsx` (created in v9.2-01)

## Existing Page Patterns (reference)

The files page (`files.tsx`) is the reference standard. It uses:
- Outer: `<div className="flex h-full min-h-0 flex-col">`
- Header: `<div className="bg-[var(--app-bg)]">` with `mx-auto w-full max-w-content p-3`
- Tab bar: `mx-auto w-full max-w-content` with underline indicator
- Content: `app-scroll-y flex-1 min-h-0` with `mx-auto w-full max-w-content`

SubPageLayout encapsulates the tab bar and content sections. The SessionHeader (already rendered by the router) handles the back button and safe area -- SubPageLayout does NOT duplicate those.

## Tasks

### Task 1: Migrate git.tsx to SubPageLayout

- **File:** `web/src/routes/sessions/git.tsx`
- **Action:** edit

Read `web/src/routes/sessions/git.tsx` first. The current page has:
- A toolbar row (Fetch/Pull/Push/Commit/Clone buttons) at lines 97-130
- An inline tab bar at lines 138-153 using `border-b-2` style
- Content area at lines 155-160

Restructure to use SubPageLayout:

1. Add import: `import { SubPageLayout } from '@/components/ui/SubPageLayout'`
2. Replace the outer `<div className="h-full flex flex-col">` with `<SubPageLayout>`:
   - `tabs` prop: the existing `tabs` array (status/history/branches/remotes)
   - `activeTab` prop: `activeTab` state
   - `onTabChange` prop: `setActiveTab`
   - `toolbar` prop: the Git action toolbar (Fetch/Pull/Push/Commit/Clone) + the fetchResult/pushPullError notification bar
   - `children`: the content rendering block (activeTab switch)
3. Remove the inline tab bar rendering (the `<div className="flex border-b...">` block at lines 138-153) -- SubPageLayout handles this
4. Remove the inline toolbar div -- it goes into the `toolbar` slot
5. Keep all dialog components (GitCloneDialog, GitPushDialog, GitPullDialog, GitCommitDialog) outside SubPageLayout at the same level

The toolbar slot should contain:
```tsx
toolbar={
  <>
    <div className="flex items-center gap-1">
      {/* Fetch button */}
      {/* Pull button */}
      {/* Push button */}
      {/* Commit button */}
      {/* Clone button */}
    </div>
    {(pushPullError || fetchResult) && (
      <div className={`text-xs mt-1 ${...}`}>
        {pushPullError || fetchResult?.msg}
      </div>
    )}
  </>
}
```

The content children should be the activeTab conditional renders without any wrapping scroll div (SubPageLayout provides the scroll wrapper).
