## Color

Uses the existing `var(--app-*)` color system from `web/src/index.css`. This phase replaces all `var(--hp-*)` references in components with their `var(--app-*)` equivalents.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--app-bg)` | Page background, header background |
| Secondary (30%) | `var(--app-secondary-bg)` | Cards, file rows, list items, hover surfaces |
| Accent (10%) | `var(--app-link)` | Active tab underline, active tab text, branch name highlight, primary action buttons |
| Destructive | `var(--app-danger)` | Error text, delete/uninstall actions, rejection buttons |
| Muted text | `var(--app-hint)` | Inactive tab text, metadata labels, secondary descriptions |
| Foreground | `var(--app-fg)` | Primary text, active element text |
| Border | `var(--app-border)` | Tab bar bottom border, card borders, dividers |
| Success | `--app-success` (NEW) | Branch current indicator, staged file badge, approval actions |
| Success subtle | `--app-success-subtle` (NEW) | Success badge background, staged file badge background |
| Warning | `--app-warning` (NEW) | Modified file status badge |
| Warning subtle | `--app-warning-subtle` (NEW) | Warning badge background |
| Primary subtle | `--app-primary-subtle` (NEW) | Branch badge background, current branch row highlight |
| Text secondary | `--app-text-secondary` (NEW) | Secondary text between hint and foreground |

Accent (`var(--app-link)`) reserved for:
- Active tab text color and underline indicator
- Primary action buttons (Commit, Clone, Search, Checkpoint)
- Branch name highlight in status panel
- Focus ring on interactive elements

Destructive (`var(--app-danger)`) reserved for:
- Error message text
- Delete branch button
- Uninstall/reject action buttons
- Undo confirmation button

### Inline Style Conversion Pattern

Convert `style={{ color: 'var(--hp-text-primary)' }}` to Tailwind class `text-[var(--app-fg)]`.

Convert `style={{ background: 'var(--hp-surface-1)' }}` to Tailwind class `bg-[var(--app-secondary-bg)]`.

Convert multi-property styles like:
```tsx
style={{ color: 'var(--hp-danger)', background: 'var(--hp-danger-subtle)' }}
```
to:
```tsx
className="text-[var(--app-danger)] bg-[var(--app-badge-error-bg)]"
```

### Pill-to-Tab Conversion

Changes/Timeline/Undo pages currently use pill-style filter buttons. These convert to the standard underline tab pattern.

**Current (pill):**
```tsx
className={`px-2 py-1 text-xs rounded-md ${
  active ? 'bg-[var(--app-link)] text-white' : 'bg-[var(--app-secondary-bg)] text-[var(--app-hint)]'
}`}
```

**Target (underline tab via SubPageLayout):**
```tsx
<SubPageLayout
  tabs={[
    { id: 'all', label: 'All' },
    { id: 'pending', label: 'Pending' },
    ...
  ]}
  activeTab={filter}
  onTabChange={setFilter}
>
  {children}
</SubPageLayout>
```

### Changes page filter conversion specifics

The Changes page has a combined filter row + bulk action row. After conversion:
- Filter pills become SubPageLayout tabs (all/pending/approved/rejected)
- Bulk action buttons (Approve All / Reject All) move into the `toolbar` slot
- Context bar remains as a separate element above the tab bar

### Timeline page filter conversion specifics

The Timeline page has 7 filter options (all, message, tool_use, file_change, summary, checkpoint, error). These become SubPageLayout tabs. The "Create Checkpoint" button moves into the `toolbar` slot on the right side.

### Undo page specifics

The Undo page uses pill-style scope selector (session/step/file) but these are not navigational tabs -- they are form controls. Conversion approach:
- Scope selector remains as styled radio buttons (keep pill style for form inputs)
- Wrap entire content in SubPageLayout (no tabs, just max-w-content + scroll)
- Convert all `var(--hp-*)` references

## Interaction Contract

### Tab switching

```
User clicks a tab:
  SubPageLayout calls onTabChange(tabId)
  Parent component updates activeTab state
  Content area re-renders for new tab
  No animation on content change (instant swap)
  Tab underline transitions via CSS transition-colors (150ms)
```

### Toolbar actions (Git page)

```
Git toolbar contains: Fetch | Pull | Push | Commit | Clone
- Fetch: calls handleFetch, shows inline result for 4s
- Pull: opens GitPullDialog
- Push: opens GitPushDialog
- Commit: opens GitCommitDialog (disabled when changedFiles === 0)
- Clone: opens GitCloneDialog
All dialog components are independent overlays, not affected by SubPageLayout.
```

### Content scroll

```
SubPageLayout content area uses app-scroll-y:
  - overflow-y: auto
  - overscroll-behavior-y: contain
  - -webkit-overflow-scrolling: touch
  - touch-action: pan-y
  - ::after pseudo-element for safe-area-inset-bottom padding
```
