# Phase v9.2: 全局子页面布局统一 - Context

**Gathered:** 2026-06-03
**Status:** Ready for planning

## Phase Boundary

提取共享 SubPageLayout 组件（带 Tab 栏插槽），统一所有子页面的 CSS 变量、内容宽度（max-w-content）、Tab 指示器样式（下划线）和 padding。全量替换 `var(--hp-*)` 为 `var(--app-*)` 等效变量（含 Editor、DiffView、SessionList、共享 UI 组件）。涉及文件：子页面路由 5 个、Git 子组件 6 个、共享 UI 组件 4 个、Editor 组件 2 个、CSS 文件 2 个。

## Implementation Decisions

### CSS 变量替换

- **D-01:** 全量替换 `var(--hp-*)` 为 `var(--app-*)` 等效变量，范围包括：子页面路由、Git 子组件、共享 UI 组件（ContextMenu、FileDialogs、ConfirmDialog、button.tsx）、Editor 组件（CodeEditor、DiffView）、SessionList
- **D-02:** 在 index.css 中新增 `--app-success` / `--app-success-subtle` / `--app-warning` / `--app-warning-subtle` / `--app-primary-subtle` / `--app-text-secondary` 变量补全映射
- **D-03:** 保留 `--hp-*` 设计令牌层（tokens.css 的 spacing/radius/shadow/duration、typography.css 的 font/text/leading/tracking/weight）——这些是基础设施级别，组件不直接引用
- **D-04:** `--hp-*` → `--app-*` 映射关系：
  - `--hp-danger` → `--app-danger`
  - `--hp-danger-subtle` → `--app-badge-error-bg`（或新增 `--app-danger-subtle`）
  - `--hp-success` → 新增 `--app-success`
  - `--hp-success-subtle` → 新增 `--app-success-subtle`
  - `--hp-warning` → 新增 `--app-warning`
  - `--hp-warning-subtle` → 新增 `--app-warning-subtle`
  - `--hp-primary` → `--app-link`
  - `--hp-primary-subtle` → 新增 `--app-primary-subtle`
  - `--hp-primary-text` → `--app-button-text`
  - `--hp-surface-0` → `--app-bg`
  - `--hp-surface-1` → `--app-secondary-bg`
  - `--hp-surface-2` → `--app-subtle-bg`
  - `--hp-text-primary` → `--app-fg`
  - `--hp-text-secondary` → 新增 `--app-text-secondary`
  - `--hp-text-tertiary` → `--app-hint`
  - `--hp-border` → `--app-border`

### SubPageLayout 组件

- **D-05:** 创建 `SubPageLayout` 共享组件（路径 `web/src/components/ui/SubPageLayout.tsx`），提供 max-w-content 包裹 + Tab 栏插槽
- **D-06:** SubPageLayout 接受 tabs / activeTab / onTabChange props，渲染统一的下划线 Tab 栏
- **D-07:** SubPageLayout 提供 children 插槽用于页面内容区域

### 内容宽度

- **D-08:** 所有子页面统一使用 `max-w-content`（960px 默认，xl+ 90%/1200px）限制内容宽度

### Tab 样式

- **D-09:** 所有子页面统一为下划线 Tab 样式（参考文件页标准），包含：绝对定位下划线、`text-sm`、`text-[var(--app-link)]` active、`text-[var(--app-hint)]` inactive
- **D-10:** Changes/Timeline/Undo 页面的 pill 按钮筛选栏转换为 Tab 下划线样式
- **D-11:** SubPageLayout 组件统一渲染 Tab 栏，各页面不再自行渲染

### Claude's Discretion

- SubPageLayout 的具体 props 接口设计（是否包含 toolbar/action slot）
- 新增 `--app-*` 变量的具体值（参考现有 --hp-* 变量的 oklch 值或使用 --app-* 系统的 RGB/十六进制风格）
- 各页面迁移到 SubPageLayout 的具体实现方式
- Git 子组件中 inline style 改为 Tailwind 类名的方式
- Tab 下划线切换动画细节

## Existing Code Insights

### Reusable Assets

- `max-w-content` CSS 类 — 已在 index.css 中定义（960px 默认，xl+ 90%/1200px），文件页标准
- `var(--app-*)` CSS 变量体系 — 已有 ~60 个变量，需补全 success/warning/primary-subtle/text-secondary
- Tab 下划线样式 — Git 页和 Extensions 页已有 `border-b-2 border-[var(--app-link)]` 实现，可直接提取为 SubPageLayout
- Git 专用颜色变量 — `--app-git-staged-color` / `--app-git-unstaged-color` / `--app-git-deleted-color` / `--app-git-renamed-color` / `--app-git-untracked-color` 已存在

### Established Patterns

- `flex h-full min-h-0 flex-col` — 所有子页面外层容器标准（v9.1 已统一）
- `border-b border-[var(--app-border)]` — Tab 栏底部边框标准
- `app-scroll-y` — 内容区滚动样式标准
- inline style `style={{ color: 'var(--hp-*)' }}` — Git 子组件大量使用，需转换为 Tailwind 类名

### Integration Points

- `web/src/router.tsx` — SessionDetailRoute 已渲染 SessionHeader，SubPageLayout 在 Outlet 层级
- 各子页面路由 — 需包裹 SubPageLayout 组件
- Git 对话框组件（GitCommitDialog、GitPushDialog、GitPullDialog） — 独立弹窗，不受 SubPageLayout 影响，但需 CSS 变量替换

## Specific Ideas

- SubPageLayout Tab 栏应提供统一的绝对定位下划线样式，而非各页面自行用 `border-b-2` 实现
- Git 子组件的 inline style 应尽量转换为 Tailwind 类名，保持代码一致性
- 新增 `--app-*` 变量值应与现有 `--app-*` 系统的配色风格一致（使用 hex/rgba 而非 oklch）
