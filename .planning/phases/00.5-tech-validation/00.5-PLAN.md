**type:** spike
**autonomous:** true

**Steps:**

**Success Criteria:**
- 确认 rlimit 方案可行或确定替代方案

### Task 6: Path Security Enhancement PoC
**type:** spike
**autonomous:** true

验证增强版路径安全中间件能防御各种路径遍历攻击。

**Steps:**
1. 创建 `scripts/poc/path-security-poc.ts`
2. 实现增强版 `sanitizePath`：
   ```typescript
   function sanitizePath(root: string, input: string): string {
     // 1. URL 解码
     const decoded = decodeURIComponent(input)
     // 2. Unicode NFC 正规化
     const normalized = decoded.normalize('NFC')
     // 3. 路径规范化
     const resolved = path.resolve(root, normalized)
     // 4. 符号链接解析
     const real = fs.realpathSync(resolved)
     // 5. 前缀检查
     if (!real.startsWith(root + path.sep) && real !== root) {
       throw new Error('Path traversal detected')
     }
     return real
   }
   ```
3. 测试攻击向量：
   - `../../../etc/passwd`
   - `%2e%2e%2f%2e%2e%2f` （URL 编码）
   - `..%c0%af..%c0%af` （Unicode 等价字符）
   - 符号链接指向外部目录
   - Windows 保留名称（`CON`、`AUX`、`COM1`）
   - 空字节注入（`file.txt%00.exe`）
4. 记录每个攻击向量的防御结果

**Success Criteria:**
- 所有已知攻击向量被成功拦截
- 合法路径正常通过
- 无误报（正常路径不被拒绝）

**files_created:**
- `scripts/poc/path-security-poc.ts`
