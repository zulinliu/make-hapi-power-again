# Phase v9.1: SessionHeader 响应式工具栏 - Context

**Gathered:** 2026-06-03
**Status:** Ready for planning

## Phase Boundary

将 SessionHeader 升级为共享布局组件，包裹所有会话子路由（聊天/Git/扩展/文件/大纲等）。桌面端在 Header 中直接显示 Files/Git/Extensions/Outline 图标（lg 断点 1024px+），移动端保持"..."菜单入口。消除桌面端图标与菜单的功能重复。子页面不再有独立 header。

## Implementation Decisions

### 架构决策

- **D-01:** SessionHeader 升级为共享布局组件，包裹所有 `/sessions/$sessionId/*` 子路由
- **D-02:** 子页面（Git/扩展/文件/变更/时间线/撤销等）移除各自独立的 header 组件，仅保留内容区域（含 tab 栏）
- **D-03:** 子页面的 tab 栏（如 Git 的 Status/History/Branches）保留在内容区顶部，不在共享 Header 中
- **D-04:** v9.1 一步到位：架构改造 + 基础样式（使用 `var(--app-*)` CSS 变量），v9.2 仅做微调

### 响应式行为

- **D-05:** 工具图标响应式断点保持 `lg` (1024px)，与当前 `hidden lg:flex` 一致
- **D-06:** 桌面端 (>=1024px): Files/Git/Extensions/Outline 图标直接显示在 SessionHeader 中
- **D-07:** 移动端 (<1024px): 工具图标隐藏，仅通过"..."菜单访问（保持现有行为）
- **D-08:** 桌面端"..."菜单仅保留会话管理功能（重命名、归档/删除），移除工具入口
- **D-09:** 移动端"..."菜单保持不变（工具 + 会话管理）

### 活跃状态与导航

- **D-10:** 活跃图标使用 `var(--app-link)` 颜色高亮，通过 TanStack Router 路由自动检测当前所在页面
- **D-11:** Toggle 导航行为：点击已激活的图标（如 Git）切换回聊天视图；点击未激活图标导航到对应页面
- **D-12:** Files 图标始终可见（与当前行为一致），其余图标 lg+ 可见

### Claude's Discretion

- 共享 Header 的具体组件结构（是否拆分为 SessionHeader + SubPageLayout）
- 路由层级重构的具体实现方式（layout route vs wrapper component）
- 过渡动画的细节
- Tab 栏在内容区的具体样式调整

- `SessionHeader` 组件：已有 Files/Git/Extensions/Outline 图标渲染逻辑和 `hidden lg:flex` 响应式类
- `SessionActionMenu` 组件：已有工具入口列表和会话管理入口，可直接改造

- `max-w-content` 包裹模式：文件页标准，所有子页面需统一

- 返回按钮模式：`h-8 w-8 rounded-full` 图标按钮

### Integration Points

- SessionHeader props — 需要从回调模式改为路由感知模式（或保留回调 + 添加路由检测）
