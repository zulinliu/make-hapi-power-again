# Phase v9.1 -- UI Design Contract

> Visual and interaction contract for the SessionHeader responsive toolbar. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, icon button size on sm+ |
| md | 16px | Default element spacing, header padding `p-3` (12px used historically, not a token) |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- `p-3` (12px) for header padding -- established pattern from files.tsx reference standard
- `gap-1.5` (6px) for metadata line spacing -- existing SessionHeader pattern
- `gap-2` (8px) for header row items -- existing SessionHeader pattern

## Color

Uses the existing `var(--app-*)` color system from `web/src/index.css`. Two parallel token systems exist (`--app-*` runtime and `--hp-*` design tokens); this phase uses `--app-*` exclusively.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--app-bg)` | Header background, content surfaces |
| Secondary (30%) | `var(--app-secondary-bg)` | Icon button hover background, subtle surfaces |
| Accent (10%) | `var(--app-link)` | Active icon highlight, tab underline, focus ring |
| Destructive | `var(--app-danger)` (#DA3633 light / #f85149 dark) | Archive/Delete actions in menu |
| Muted text | `var(--app-hint)` | Inactive icon color, metadata text |
| Foreground | `var(--app-fg)` | Active text, icon hover state |
| Border | `var(--app-border)` | Header bottom border, dividers |

Accent (`var(--app-link)`) reserved for:
- Active tool icon color when current route matches that tool
- Tab indicator underline
- Focus-visible ring on interactive elements
- Sidebar resize handle on hover/drag

Destructive (`var(--app-danger)`) reserved for:
- Archive session menu item (when session is active)
- Delete session menu item (when session is archived)

## Component Inventory

### SessionHeader (upgraded to shared layout)

**Structure:**
```
<div bg-[--app-bg] pt-[env(safe-area-inset-top)]>
  <div max-w-content flex items-center gap-2 p-3>
    [Back button] [Session info] [Files icon] [Git icon] [Extensions icon] [Outline icon] [...] button
  </div>
</div>
```

**Icon button states:**

| State | Classes |
|-------|---------|
| Default | `flex h-10 w-10 items-center justify-center rounded-full text-[var(--app-hint)] transition-colors sm:h-8 sm:w-8` |
| Hover | `hover:bg-[var(--app-secondary-bg)] hover:text-[var(--app-fg)]` |
| Active (route matched) | `text-[var(--app-link)]` replaces `text-[var(--app-hint)]` -- no background change |
| Focus-visible | `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-[var(--app-link)]` |
| Mobile-hidden (Git/Extensions/Outline) | Append `hidden lg:flex` |
| Always-visible (Files, ..., Back) | No responsive hide class |

**Responsive breakpoint:** `lg` (1024px) -- consistent with existing `hidden lg:flex` pattern.

**Active state detection:** Use TanStack Router's `useMatch` or `useLocation` to determine if current route matches the tool's target route. Apply `text-[var(--app-link)]` class instead of `text-[var(--app-hint)]`.

**Toggle navigation (D-11):**
- Click active icon: navigate back to chat view (`/sessions/$sessionId`)
- Click inactive icon: navigate to that tool's route
- This matches VS Code sidebar toggle behavior

### SessionActionMenu (responsive content)

**Desktop (>=1024px):**
- "Tools" section header and Git/Extensions/Outline items are hidden
- Only session management items visible: Rename, Archive/Delete
- Menu triggered by "..." button click

**Mobile (<1024px):**
- "Tools" section header and all tool items visible (current behavior)
- Session management items visible below
- All current functionality preserved

**Implementation:** Use `window.matchMedia('(min-width: 1024px)')` or CSS `hidden lg:flex` / `lg:hidden` within the menu render to conditionally show/hide the tools section.

### Sub-page content area

**After removing individual sub-page headers, each sub-route renders only:**

```
<div flex h-full min-h-0 flex-col>
  [Tab bar if applicable] -- already in content area
  [Scrollable content] -- app-scroll-y flex-1 min-h-0
</div>
```

Sub-pages no longer render their own back button, title bar, or `pt-[env(safe-area-inset-top)]` -- the shared SessionHeader handles all of that.

## Layout Contract

### SessionHeader dimensions

| Property | Value |
|----------|-------|
| Outer padding-top | `env(safe-area-inset-top)` via outer div |

1. Back button (always)
2. Session title + metadata (flex-1)

4. Git icon (lg+ only, hidden lg:flex)
5. Extensions icon (lg+ only, hidden lg:flex)
6. Outline icon (lg+ only, hidden lg:flex)
7. "..." menu button (always)

## Interaction Contract

### Toggle navigation behavior (D-11)

```
User clicks Git icon:
  IF current route is /sessions/$sessionId/git:
    navigate to /sessions/$sessionId (chat view)
  ELSE:
    navigate to /sessions/$sessionId/git

Same logic for Extensions, Files, Outline.
```

### Menu behavior

```
Desktop (>=1024px):
  "..." menu shows: Rename, Archive/Delete
  Tools section hidden

Mobile (<1024px):
  "..." menu shows: [Tools heading] Git, Extensions, Outline [divider] [Session heading] Rename, Archive/Delete
  All current items preserved
```

### Responsive transition

- Use CSS-only `hidden lg:flex` / `lg:hidden` for icon visibility
- No JavaScript-driven class toggling for layout
- No transition animation on the icon appear/disappear (would cause layout shift)
- The "..." menu position recalculates on resize (existing behavior in SessionActionMenu)
