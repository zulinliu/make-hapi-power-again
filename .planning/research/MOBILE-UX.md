## 1. 移动终端 UX

### 参考应用

#### Termius

- **平台:** iOS / Android / macOS / Windows / Linux
- **用户群:** 2M+ 工程师，SOC 2 Type II 合规
- **截图描述:**
  - 深色终端背景，底部 Tab 导航（Hosts / Keys / Snippets / Port Forwarding / Terminal）
  - 终端视图顶部显示连接标签页（可左右滑动切换），右上角键盘工具栏
  - SFTP 文件浏览器并排显示本地和远程文件
  - Vault 加密同步界面：设备列表 + 加密状态指示

**关键交互模式:**
- **Vault 系统:** E2E 加密的跨设备凭证同步（AES-256），团队可共享连接配置
- **IDE 风格自动补全:** 输入命令时弹出下拉建议，类似 VS Code 的 IntelliSense
- **工作区恢复:** 重开后自动恢复标签页分组和分屏布局
- **终端多人协作 (Terminal Multiplayer):** 实时共享终端会话，多用户可同时操作
- **代码片段 (Snippets):** 常用命令保存为可复用片段，支持变量替换
- **FIDO2 + 生物识别:** 支持硬件安全密钥和 Touch ID/Face ID 认证
- **Gloria AI DevOps Agent:** 内置 AI 助手执行运维任务
- **分屏终端:** iPad 支持并排两个终端会话

#### Blink Shell

- **平台:** iOS / iPadOS / macOS
- **定价:** $19.99/年（开源核心）
- **截图描述:**
  - 主界面为纯黑背景终端，左侧抽屉显示服务器列表和最近连接
  - 分屏视图：左侧 ssh 会话 + 右侧 mosh 会话
  - Blink Code 界面：内嵌 VS Code Web 编辑器
  - 键盘定制面板：修饰键映射表（Caps→Esc, Cmd→Ctrl 等）

**关键交互模式:**
- **Mosh 协议:** 自动重连，支持网络切换（WiFi↔蜂窝）和设备睡眠唤醒
- **本地回显 (Local Echo):** 消除网络延迟，输入字符立即显示
- **Blink Code:** 内置 VS Code Web 连接，支持 Codespaces / GitPod / 自建服务器
- **外部键盘深度定制:** Caps Lock → Esc, Cmd → Ctrl, 完全可编程的修饰键映射
- **主题/字体画廊:** 内置 Pragmata Pro 等开发者字体，社区主题分享
- **Air Terminals:** 利用 Apple TV / 4K 显示器扩展终端输出
- **iOS Files.app 集成:** SFTP 挂载到 Files.app，可被其他 app 直接访问
- **内建 Unix 工具:** vim, grep, curl, scp, sftp 等开箱即用

#### a-Shell

- **平台:** iOS / iPadOS
- **定价:** 免费，开源
- **截图描述:**
  - 多窗口终端，每个窗口有独立的上下文、历史和外观配置
  - `config` 命令调出设置面板：字体大小、配色方案、光标样式
  - Python REPL 界面，可直接运行脚本
  - Apple Shortcuts 集成流程图

**关键交互模式:**
- **多窗口管理:** 每个窗口独立的上下文、命令历史、外观、工作目录
- **多语言 REPL:** 内置 Python 3.11、Lua、JavaScript 运行时
- **WebAssembly 编译:** C/C++ 程序编译为 WASM 在本地执行
- **iOS 沙盒目录访问:** 通过 bookmarks 机制持久化访问权限
- **Apple Shortcuts 集成:** 提供 Execute Command / Put File / Get File 三个 Action
- **VoiceOver 支持:** 终端内容可通过屏幕阅读器朗读
- **TeX 排版:** 内置 TeX 引擎，支持文档编译

## 3. 触摸手势系统

### 参考应用和模式

#### 原生手势规范

| 平台 | 手势 | 用途 |
|------|------|------|
| iOS | 从左边缘右滑 | 返回上一页 |
| iOS | 从底部上滑 | 回到主屏幕 |
| iOS | 底部横条左右滑动 | 切换最近应用 |
| Android | 系统返回手势 | 返回上一页 |
| 通用 | 长按 | 上下文菜单 |
| 通用 | 双指捏合 | 缩放 |
| 通用 | 双指张开 | 放大 |
| 通用 | 单指下拉 | 刷新/滚动 |

#### 终端/IDE 中的手势应用

| 应用 | 手势 | 行为 |
|------|------|------|
| Termius | 三指轻触 | 显示/隐藏键盘工具栏 |
| Blink Shell | 双指滑动 | 终端内滚动 |
| Blink Shell | 双指捏合 | 调整字体大小 |
| GitHub Mobile | 左滑通知 | 快速操作（已读/静音/完成） |
| GitHub Mobile | 右滑 PR 评论 | 回复评论 |
| Working Copy | 下拉文件列表 | 刷新 Git 状态 |
| a-Shell | 双指轻触 | 粘贴 |

### 手势冲突处理

移动端手势最大的挑战是与系统手势的冲突：

1. **边缘滑动 vs 系统返回:** 避免在距屏幕边缘 20px 内放置自定义手势
2. **下拉刷新 vs 滚动:** 使用阈值区分（快速下拉 = 刷新，持续下拉 = 滚动）
3. **长按 vs 上下文菜单:** 300ms 以上触发，提供触觉反馈
4. **双指操作 vs 缩放:** 终端/编辑器中双指操作默认为缩放，需明确切换模式

### Hapi Power 建议

**v0.1 手势集（最小可用）:**

| 手势 | 区域 | 行为 |
|------|------|------|
| 点击 | 文件卡片 | 展开 diff 详情 |
| 左滑 | 审查卡片 | 拒绝（红色指示） |
| 右滑 | 审查卡片 | 批准（绿色指示） |
| 长按 | 文件/文本 | 上下文菜单（复制/分享） |
| 下拉 | 列表顶部 | 刷新 |
| 双指捏合 | 终端/代码 | 调整字体大小 |
| 双指滑动 | 终端 | 滚动输出 |

**v0.2 手势增强:**

| 手势 | 区域 | 行为 |
|------|------|------|
| 双指上滑 | 终端 | 滚动到最新输出 |
| 三指轻触 | 全局 | 切换键盘工具栏 |
| 左右滑动 | 终端工具栏 | 切换会话标签页 |
| 捏合放手 | 代码 | 重置缩放 |

**实现建议:**
- 使用 `touch-action` CSS 属性控制浏览器默认手势
- 边缘 20px 内不响应自定义手势（避免与系统返回手势冲突）
- 所有手势操作提供触觉反馈（`navigator.vibrate(10)`）
- 手势识别使用 `pointer events`（统一触摸和鼠标输入）
- 考虑使用 `use-gesture` 库（React 生态成熟的手势库）

## 4. 虚拟键盘适配

### 参考应用

#### Blink Shell 键盘方案

**外部键盘深度定制:**
- Caps Lock → Esc（vim 用户刚需）
- Cmd → Ctrl（macOS 习惯适配终端）
- 完全可编程的修饰键映射表
- 快捷键自定义面板

**虚拟键盘工具栏:**
- 终端专用工具栏位于键盘上方，包含：
  - **修饰键:** Ctrl、Alt、Esc、Tab（锁存式，点击高亮，再按下一个键时组合触发）
  - **方向键:** 上下左右箭头
  - **特殊键:** Pipe（|）、Tilde（~）、Backtick（`）、Dollar（$）
  - **自定义:** 用户可将常用命令绑定为工具栏按钮

#### Termius 键盘方案

- 顶部工具栏：Tab / Ctrl / Esc / 方向键
- 工具栏可上下滑动展开更多按键
- 长按工具栏按键显示变体（如长按 `.` 显示 `.` / `..` / `...` / `/`）
- Snippets 快速插入：工具栏上的闪电图标弹出命令片段列表

#### a-Shell 键盘方案

- iOS 系统键盘 + 自定义输入 accessory view
- 支持硬件键盘连接自动隐藏虚拟键盘
- `config` 命令切换键盘布局偏好

### 移动端键盘的关键问题

| 问题 | 影响 | 解决方案 |
|------|------|----------|
| 虚拟键盘遮挡内容 | 终端输出、代码编辑区被遮挡 | `visualViewport` API 动态调整布局 |
| 缺少修饰键 | 无法输入 Ctrl+C、Tab 等终端常用键 | 自定义工具栏提供修饰键 |
| 按键延迟 | 终端交互卡顿 | 本地回显 + 防抖 |
| 旋转切换 | 键盘状态丢失 | 保存/恢复输入状态 |
| 自动纠正 | 代码被自动"修正" | 输入框设置 `autocomplete="off"` `autocorrect="off"` `autocapitalize="off"` `spellcheck="false"` |
| 蓝牙键盘兼容性 | 修饰键行为不一致 | 键盘事件标准化处理 |

### Hapi Power 建议

**v0.1 键盘方案（只读终端，键盘需求低）:**

由于 v0.1 移动端终端为只读（AIWF-10），虚拟键盘适配主要影响：
- 搜索框：标准文本输入，`autocorrect="off"` `autocapitalize="off"`
- 审查评论：标准文本域，支持基本 Markdown
- 拒绝理由：简单文本输入

```html
<!-- 移动端输入框属性 -->
<input
  type="text"
  autocomplete="off"
  autocorrect="off"
  autocapitalize="off"
  spellcheck="false"
  enterkeyhint="search"
/>
```

**v0.2 键盘增强（MOB-01 终端触摸优化）:**

```
┌──────────────────────────────┐
│         终端输出区域           │
│                              │
│  $ npm run build             │
│  Building... done in 2.3s    │
│                              │
├──────────────────────────────┤
│  Tab │ Esc │ Ctrl│ Alt│  ↑↓←→│  ← 终端工具栏
├──────────────────────────────┤
│         虚拟键盘              │
└──────────────────────────────┘
```

- 修饰键采用**锁存模式:** 点击 Ctrl（高亮），再按 C = 发送 Ctrl+C
- 方向键为紧凑型四向按钮
- 常用符号快速访问栏: `|` `~` `` ` `` `$` `/` `\` `&` `;`
- 工具栏高度约 40px，背景与终端背景一致
- 使用 `visualViewport` API 动态调整终端高度

**CSS 关键属性:**
```css
.terminal-viewport {
  height: calc(100dvh - var(--toolbar-height, 0px));
  /* 使用 dvh 替代 vh，自动适应虚拟键盘弹出 */
}

.terminal-container {
  touch-action: pan-y;
  /* 允许纵向滚动，禁用浏览器默认缩放 */
}

@media (max-height: 400px) {
  /* 虚拟键盘弹出时 */
  .terminal-toolbar {
    display: none; /* 隐藏自定义工具栏，避免双重遮挡 */
  }
}
```

## 5. 移动端 AI 聊天界面

### 参考应用

#### ChatGPT Mobile

- **平台:** iOS / Android
- **截图描述:**
  - 对话列表（左侧抽屉）+ 当前对话（主区域）
  - 消息气泡：用户消息靠右（蓝色），AI 消息靠左（白色/深色）
  - 底部输入栏：文本框 + 语音按钮 + 附件按钮 + 发送按钮
  - 语音对话模式：全屏波形动画 + 实时转录文字

**关键交互模式:**
- **语音对话:** 实时语音输入 → AI 语音回复，可随时打断
- **多模态输入:** 拍照/相册/文件附加到对话
- **对话分支:** 编辑历史消息重新生成回复
- **项目（Projects）:** 按项目组织对话和文件
- **分享对话:** 生成公开链接

#### Claude Mobile (claude.ai)

- **平台:** iOS / Android / Web
- **截图描述:**
  - 对话界面与 ChatGPT 类似，但 AI 回复更结构化（带折叠代码块）
  - Remote Control 界面：手机上显示 CLI 任务状态 + 进度条
  - Cowork 模式：手机端发起对话 → 桌面端继续

**关键交互模式:**
- **Remote Control:** 从手机发送任务给 Claude Code CLI（桌面端），实时查看执行进度
- **Cowork 配对:** 手机开始对话 → 桌面无缝接续，共享对话历史
- **桌面+移动同步:** 对话、项目、记忆、偏好全平台同步
- **Projects 管理:** 移动端可查看和管理项目上下文
- **企业部署:** 支持企业级策略和 MDM 部署

### Hapi Power 建议

**v0.1 AI 聊天移动端:**

| 查看代理状态 | 进度条 + 状态文本 | 正在编辑/正在运行命令/等待审批 |
| 审查文件变更 | 卡片列表 + 滑动审批 | AIWF-09 三态审查 |
| 查看终端输出 | 只读 200 行 | AIWF-10 实时状态 |
| 推送通知 | Web Push | AIWF-11 审批/完成通知 |
| 查看操作时间线 | 按类型过滤的列表 | AIWF-04 |

**v0.2 增强:**
- 简单指令发送：发送"停止"/"继续"/"撤销"等快捷指令
- 语音审批："批准所有变更"语音指令（AGXP-02 语音对话集成）
- 移动端发起对话：简单的文字指令输入

## 8. 响应式断点体系

### 行业标准断点

| 断点名称 | 宽度范围 | 设备 | 典型用途 |
|---------|---------|------|---------|
| xs | < 640px | iPhone SE / 小屏手机 | 单列布局，底部导航 |
| sm | 640-767px | 大屏手机（横屏）/ 小平板 | 双列紧凑布局 |
| md | 768-1023px | iPad / Android 平板 | 侧边栏 + 内容区 |
| lg | 1024-1439px | iPad Pro / 小笔记本 | 完整布局，紧凑间距 |
| xl | 1440px+ | 桌面显示器 | 完整布局，宽松间距 |

### Hapi Power 断点定义（DS-04）

```
/* Tailwind CSS 4 配置 */
@theme {
  --breakpoint-sm: 640px;
  --breakpoint-md: 1024px;
  --breakpoint-lg: 1440px;
}
```

三个核心断点：

| 断点 | 宽度 | 布局策略 |
|------|------|---------|
| Mobile | < 640px | 单列，底部 Tab 导航，/m/* 路由 |
| Tablet | 640-1024px | 两栏（侧边栏可折叠），顶部导航 |
| Desktop | > 1024px | 三栏（导航 + 主内容 + 面板），侧边栏导航 |

### CSS 实现策略

```css
/* 基础移动优先 */
.layout {
  display: grid;
  grid-template-rows: auto 1fr auto;
  grid-template-columns: 1fr;
  height: 100dvh;
}

/* 平板端 */
@media (min-width: 640px) {
  .layout {
    grid-template-columns: 240px 1fr;
    grid-template-rows: auto 1fr;
  }
}

/* 桌面端 */
@media (min-width: 1024px) {
  .layout {
    grid-template-columns: 56px 240px 1fr;
  }
}

/* 安全区域适配（iOS PWA） */
.layout {
  padding-top: env(safe-area-inset-top);
  padding-bottom: env(safe-area-inset-bottom);
  padding-left: env(safe-area-inset-left);
  padding-right: env(safe-area-inset-right);
}
```

**关键 CSS 注意事项:**
- 使用 `100dvh` 替代 `100vh`（动态视口高度，适应虚拟键盘弹出）
- 使用 `env(safe-area-inset-*)` 适配 iOS 刘海/底部手势条
- 使用 `touch-action` 控制触摸行为，避免浏览器默认手势冲突
- 使用 `overscroll-behavior: contain` 防止过度滚动传播
- 字体大小最小 14px（移动端可读性底线）

### 技术栈建议

| 领域 | 推荐方案 | 理由 |
|------|---------|------|
| 手势识别 | `@use-gesture/react` | React 生态最成熟，支持触摸/鼠标统一 |
| 触觉反馈 | `navigator.vibrate()` | Web 标准 API，iOS/Android 均支持 |
| 视口适配 | `visualViewport` API | 准确响应虚拟键盘弹出/收起 |
| 安全区域 | `env(safe-area-inset-*)` | iOS PWA 必需 |
| 滑动操作 | 自实现（CSS transform + transition） | 审查卡片的滑动审批交互 |
| 推送通知 | Web Push API + Service Worker | AIWF-11，iOS 16.4+ 支持 |
| PWA | `vite-plugin-pwa` (injectManifest) | Vite 生态，离线缓存 + 推送 |
| 视口单位 | `dvh` (dynamic viewport height) | 自动适应浏览器 UI 变化 |

### 设计原则

1. **移动端不是桌面的缩小版** — 重新设计交互模式，而非缩放布局
2. **只读优先，安全第一** — v0.1 移动端终端只读，避免误操作风险
3. **触摸目标 ≥ 44px** — Apple HIG 推荐，避免误触
4. **离线降级** — 网络中断时显示缓存内容 + 离线提示
5. **性能预算** — 移动端 JavaScript 包体积 < 300KB gzipped
6. **渐进增强** — 基础功能不依赖 JavaScript 高级特性
