# Product

## Register

product

## Users

开发者和 AI 编码代理使用者。他们在浏览器中使用 Hapi Power 管理远程服务器上的代码项目、文件和开发环境。核心场景：在 iOS 手机上浏览服务器文件、编辑代码、克隆项目、管理项目结构，然后启动 AI 会话进行开发。他们同时使用桌面端浏览器进行更复杂的工作。移动端使用场景多在碎片时间快速查看和轻量操作。

## Product Purpose

Hapi Power 的全局文件管理器让用户在浏览器中直接管理远程服务器上的完整文件系统。不只是一个目录选择器，而是一个功能完备的文件管理工具：浏览、搜索、编辑、传输、版本控制全部在一个页面完成。iOS 移动端体验优先，桌面端分栏布局无缝衔接。成功标准：用户可以在手机上完成 80% 的日常文件管理操作而不感到受限。

## Brand Personality

**有力、精确、温暖**。电光橙品牌色传递能量和活力，"力量几何"设计语言传达专业和精确，橙色调中性色传递温暖而非冷漠的技术感。整体感觉：一个你愿意信赖的开发伙伴，而不是冰冷的工具。

## Anti-references

- 不要做成 VS Code 文件管理器的简化版：那是桌面端代码编辑器的思维，不是移动端文件管理器的思维
- 不要做成 cPanel / 宝塔面板：那类面板过于笨重，充斥着表单和配置项
- 不要做成 Windows Explorer 的 Web 版：目录树的交互模式不适合移动端
- 避免长按、右键为唯一操作入口的设计：移动端必须有明确可见的操作按钮
- 避免灰色文字、低对比度、信息密度过低的浪费空间设计

## Design Principles

1. **触控优先，点击即达**：每个操作最多两步完成，操作按钮始终可见
2. **层级清晰，空间感知**：用户始终知道自己在哪、能去哪、怎么回来
3. **速度感知，状态透明**：每个操作有即时反馈，进度可见，结果明确
4. **移动端不妥协**：移动端不是缩小版桌面端，而是精心设计的独立体验
5. **一个页面，完整能力**：不需要跳转到其他页面就能完成文件管理的所有操作

## Accessibility & Inclusion

- WCAG 2.1 AA 级别
- 触控目标最小 44x44px（iOS HIG 标准）
- 对比度满足 4.5:1（正文）和 3:1（大文本）
- 支持减少动画偏好（prefers-reduced-motion）
- 键盘导航支持（桌面端）
- 中英双语 i18n
