# Product

## Register

product

## Users

开发者和 AI 编码代理使用者。他们用 Hapi Power 在浏览器和 iOS PWA 中管理 AI 编码会话、远程机器、代码文件、终端、Git 变更、模型供应商和项目记忆。核心场景：桌面端进行完整开发与审查，移动端在碎片时间接管会话、纠偏 Agent、检查上下文风险、查看代码变化、提交同步，并把关键对话沉淀为可复用文档。

## Product Purpose

Hapi Power 是面向 AI 编码代理的全栈开发者工作台。它把 AI 编码对话变成可驾驶、可观测、可复盘的工程闭环：接入可信模型，工作中即时纠偏，观察上下文风险，追踪每次代码变化，并把会话沉淀为项目记忆。

v0.18.0 的产品主线固定为五节点能力环：

1. **接入：模型星桥 / Model Nexus**，统一治理模型 Provider、健康检测、能力缓存和 Agent 分配矩阵。
2. **驾驶：引导光标 / Guide Beam**，在 Agent thinking 时支持“排队 / 立即引导”，纠偏不清空普通队列。
3. **观测：上下文脉冲 / Context Pulse**，用 `上下文：40%` 展示上下文占用和可靠性风险。
4. **追踪：Git 脉络 / Git Atlas**，把分支态势、变更地图、Diff、提交篮和同步风险合并为 Git 决策舱。
5. **沉淀：会话织锦 / Session Loom**，把会话大纲升级为导出、提炼、资产管理和项目记忆。

文件管理器、Monaco Editor、PTY 终端、插件与 Skill 管理、PWA 推送仍是工作台底座，但不再是单一产品定位。成功标准：用户能在一个浏览器工作台中完成从模型接入、Agent 驾驶、上下文观测、Git 追踪到会话沉淀的完整 AI 编码流程，并且移动端不显著降级。

## Brand Personality

**有力、精确、温暖**。电光橙品牌色传递能量和活力，"力量几何"设计语言传达专业和精确，橙色调中性色传递温暖而非冷漠的技术感。整体感觉：一个你愿意信赖的开发伙伴，而不是冰冷的工具。

## Anti-references

- 不要做成单纯的 Chat UI：Hapi Power 是可控工程工作台，不是只展示对话气泡。
- 不要做成 VS Code 的 Web 简化版：代码编辑只是底座，核心差异是 AI Agent 工作流闭环。
- 不要做成命令面板集合：Git、Provider、导出和上下文都必须先给状态与推荐动作，再展开细节。
- 不要把五个特色功能退回旧表达：密钥表单、旧 Git 命令页、旧上下文状态条、旧大纲下载入口都不是 v0.18 的表达。
- 避免长按、右键为唯一操作入口：移动端必须有明确可见的操作按钮。
- 避免灰色文字、低对比度、信息密度过低的浪费空间设计。

## Design Principles

1. **可驾驶**：Agent 工作中也要保留用户方向盘，关键纠偏必须可见、可反馈、可降级。
2. **可观测**：模型健康、上下文风险、Git 同步态、导出隐私状态必须用明确数据和原因表达。
3. **可追踪**：每次代码变化、提交选择、危险同步动作都要可检查、可确认、可复盘。
4. **可沉淀**：会话不是临时聊天记录，而是能转成设计方案、PRD、决策日志和经验卡的项目资产。
5. **移动端不妥协**：移动端不是缩小版桌面端，必须有 44px 触控目标、safe-area、键盘和下载/分享 fallback。
6. **一个工作台，完整闭环**：用户不需要在终端、IDE、Git GUI、导出脚本和模型配置页之间来回切换。

## Accessibility & Inclusion

- WCAG 2.1 AA 级别
- 触控目标最小 44x44px（iOS HIG 标准）
- 输入框字号不低于 16px，避免 iOS 自动缩放
- 对比度满足 4.5:1（正文）和 3:1（大文本）
- 支持减少动画偏好（prefers-reduced-motion）
- Sheet、Drawer、Popover、Dialog 必须有焦点陷阱和关闭后焦点返回
- 键盘导航支持（桌面端）
- 中英双语 i18n，新增用户可见文案必须 en / zh-CN parity
